Introduction to Version Control

Version control is a system that tracks changes to files over time. It allows developers to save different versions of a project and return to earlier states when needed.

Using version control also makes collaboration easier by allowing multiple people to work on the same codebase without conflicts.

  • Tracks changes to files
  • Prevents accidental data loss
  • Supports team collaboration
git status
Installing Git

Git must be installed before it can be used for version control. The installation process depends on your operating system.

After installation, Git can be accessed through the command line or terminal.

  • Windows users can install Git using Git Bash
  • macOS users can install Git with Homebrew
  • Linux users can install Git via package managers
git --version
Core Workflow

The core Git workflow consists of initializing a repository, staging changes, and committing those changes to the project history.

This workflow ensures that every change is intentional and recorded with a message explaining what was modified.

  • Initialize a repository
  • Stage file changes
  • Commit changes with a message
git init
git add .
git commit -m "Initial commit"
Branching and Merging

Branching allows developers to create separate lines of development without affecting the main project code.

Merging combines changes from one branch into another, typically after a feature has been completed and tested.

  • Create branches for new features
  • Switch between branches
  • Merge completed work
git branch
git checkout -b feature-branch
Working with Remotes

Remote repositories allow developers to store and share their Git projects online using platforms like GitHub.

By connecting a local repository to a remote one, changes can be pushed and pulled between team members.

  • Add a remote repository
  • Push local changes
  • Pull updates from others
git remote add origin https://github.com/user/repo.git
git push origin main
Common Mistakes and Fixes

Mistakes are common when learning Git, such as committing the wrong files or writing incorrect commit messages.

Git provides tools that allow users to undo or fix mistakes without losing their work.

  • Undo staged changes
  • Amend recent commits
  • Restore deleted files
git reset
git checkout -- filename
